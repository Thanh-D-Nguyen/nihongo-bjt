# Learning Quality Gate

Use for learner dashboard, study, flashcards, learning paths, coaching, comeback, and progress features.

## Pass checklist

- Learner knows what to do today.
- Screen does not overload the learner with too many choices.
- Mistakes produce specific remediation.
- Flashcards include useful business/context examples, not only isolated words.
- Progress measures effort and mastery, not only score.
- Copy does not shame learners who perform poorly.
- Comeback mode exists or is intentionally out of scope.
- Spacing/review logic is real and persisted.
- Learning Assist is available where Japanese text may block progress.
- Sensory/social elements do not interrupt focused study.
- Practical life topics keep study/remediation as the main action.

## Evidence

- file paths or screenshots
- API/data source for progress
- test/build command result
- known gaps and owner

## Blockers

- fake progress/coaching
- quiz/study result without remediation
- streak punishment or shame copy
- persistent progress only in local UI state
