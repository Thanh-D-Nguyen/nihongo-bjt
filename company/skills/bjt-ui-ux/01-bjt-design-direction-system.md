# BJT Design Direction System

## When to Use

Use before designing or redesigning any major learner/admin UI surface.

## Approved Direction

NihonGo BJT should feel like a serious Japanese learning console:

- focused like a study desk
- trustworthy like an exam prep product
- warm enough for daily habit
- dense enough for adult learners
- accessible for Japanese reading fatigue

## Recommended Style Mix

- base: Accessible and ethical education UI
- admin: data-dense operational dashboard
- learner dashboard: focused daily study hub
- quiz/mock exam: quiet exam console
- reading assist: paper/e-ink inspired reading layer
- progress analytics: comparative/drill-down dashboard
- postcard/share: restrained editorial visual, opt-in privacy

## Visual Tokens to Prefer

- neutral surfaces with clear contrast
- one primary action color
- restrained semantic colors for success/warning/danger/info
- 4/8px spacing rhythm
- 8px or smaller radius unless existing component requires otherwise
- readable body type for Vietnamese and Japanese
- no negative letter spacing

## Visual Effects to Prefer

- subtle hover/focus
- clear active state
- progress transition only when data changed
- inline feedback near the action
- reduced-motion alternative

## Anti-Patterns

- neon/cyberpunk/crypto visual language
- AI purple-blue gradient dominance
- playful child-learning aesthetics
- marketing landing-page heroes in app workflows
- cramped Japanese text without line height/furigana strategy
- overly calm beige UI that hides hierarchy

## Output Checklist

- chosen surface type: admin, dashboard, study, exam, reading, share, analytics
- selected style mix
- rejected style risks
- token/pattern reuse path
