# BJT Assessment and Exam UX Skill

## When to Use

Use for quiz, mock exam, scoring, result, review, remediation, and assessment analytics UI.

## Required Checks

- Exam mode and practice mode are visually distinct.
- Timed exam mode does not reveal forbidden reading/meaning help.
- Score/band is labeled estimated unless official.
- Result page explains what changed and what to study next.
- Difficulty, skill tags, and remediation are visible after answering.
- Time pressure is shown without panic design.
- Answer review never leaks correct answers before submission.

## UX Patterns

- Exam console: timer, section, progress, question, answer controls.
- Practice support: optional reading assist and explanation after answer.
- Result summary: estimated band, confidence caveat, weak skills, next review.
- Remediation card: skill tag, wrong pattern, recommended deck/path.

## Anti-Patterns

- Big celebratory UI for unreliable score.
- Official-looking BJT score claims.
- Reading assist active in strict timed exam.
- Correct-answer reveal through disabled UI/state.
- Result page with score only and no recovery plan.

## Output Checklist

- mode behavior named
- answer-key leak check recorded
- estimated score copy checked
- remediation path named
