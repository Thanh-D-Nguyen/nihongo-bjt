# BJT UI/UX Principles

## When to Use

Use for every learner-facing page, public learning page, study flow, quiz flow, onboarding flow, and admin page that affects learning quality.

This skill adapts external UI/UX inspiration into a BJT learning product. Do not clone generic style systems.

## Design Direction

- Audience: Vietnamese learners preparing for BJT and working/living in Japan.
- Tone: calm, focused, professional, encouraging.
- Layout: task-first, low distraction, clear next action.
- Visual system: restrained premium education, not entertainment-first.
- Motion: functional feedback only; no attention traps.
- Signature: Japanese text is readable, explorable, and connected to remediation.

## Required Checks

- The screen tells the learner what to do next.
- Japanese text is readable with appropriate support.
- Visual hierarchy protects concentration.
- Progress is real, explainable, and not shame-inducing.
- Media, sharing, competition, and gamification serve learning.
- Admin UX remains dense, operational, and not learner-marketing style.

## Anti-Patterns

- Generic SaaS templates without learning intent.
- Over-styled cards, gradients, glass effects, or hero sections inside study tools.
- Gamified visuals that overpower the Japanese task.
- Fake confidence, fake progress, or misleading BJT score claims.
- Decorative motion/audio/video that interrupts study.

## Output Checklist

- target learner/admin behavior named
- design direction named
- learning risk named
- relevant BJT UI/UX skills loaded
- gate evidence or blocker recorded
