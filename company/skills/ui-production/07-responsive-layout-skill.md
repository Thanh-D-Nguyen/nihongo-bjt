# Responsive Layout Skill

## When to Use

Use for every admin and learner page, especially tables, dashboards, quiz flows, flashcards, forms, and media layouts.

## Required Checks

- Desktop, tablet, and mobile layout are considered.
- Fixed-format elements have stable dimensions.
- Text does not overlap, clip, or overflow buttons/cards.
- Tables have responsive strategy: horizontal scroll, column priority, or card/list transform.
- Primary actions remain reachable.
- Sticky headers/footers do not hide content.

## Anti-Patterns

- Viewport-width font scaling.
- Buttons that shrink below readable/tappable size.
- KPI cards that wrap into broken layouts.
- Mobile table with unreadable columns.
- Hidden actions on mobile.

## Output Checklist

- responsive strategy named
- mobile/tablet/desktop notes recorded
- overflow/clipping risk checked
- touch target risk checked
