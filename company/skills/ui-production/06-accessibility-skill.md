# Accessibility Skill

## When to Use

Use for every UI change, especially forms, tables, icon buttons, media, postcards, charts, quizzes, flashcards, and navigation.

## Required Checks

- Keyboard navigation works.
- Focus visible.
- Icon buttons have accessible names.
- Semantic headings and landmarks are reasonable.
- Form controls have labels.
- Tables have headers.
- Color contrast is acceptable.
- Reduced motion is respected.
- Images/media have alt text, captions, or transcripts where applicable.
- No information is conveyed by color only.

## Anti-Patterns

- Click-only controls.
- Icon button without aria-label or visible text.
- Low contrast status badges.
- Autoplay audio/video.
- Motion that cannot be reduced.

## Output Checklist

- keyboard/focus checked
- labels/aria checked
- contrast/media notes recorded
- accessibility gaps owned
