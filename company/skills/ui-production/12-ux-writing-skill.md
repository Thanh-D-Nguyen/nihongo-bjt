# UX Writing Skill

## When to Use

Use for admin labels, learner guidance, coaching, errors, empty states, onboarding, quiz results, monetization, sharing, and legal/privacy UX.

## Required Checks

- Tone is calm, professional, and encouraging.
- Learner next action is clear.
- Mistakes turn into remediation, not shame.
- Admin copy is precise and action-oriented.
- Success messages describe what actually happened.
- Estimated scores and predictions are clearly labeled.
- Sensitive topics avoid advice claims.

## Anti-Patterns

- Childish or hype copy.
- Shame language.
- Vague "completed successfully" for complex workflows.
- Raw backend jargon.
- Overpromising BJT score outcomes.
- Dark-pattern urgency.

## Output Checklist

- target audience named
- tone risk checked
- next action clarity checked
- sensitive claim risk checked
