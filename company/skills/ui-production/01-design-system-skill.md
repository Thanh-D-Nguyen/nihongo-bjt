# Design System Skill

## When to Use

Use before creating or changing visual UI structure, components, colors, typography, icons, tables, cards, forms, or dashboards.

## Required Checks

- Reuse existing components first.
- Use design tokens or existing CSS variables/classes.
- Keep spacing, radius, shadow, border, and typography hierarchy consistent.
- Use consistent icon style and button sizing.
- Preserve established page/header/table/form/card patterns.
- Document any new pattern.
- For BJT learner UI, follow `docs/design/bjt-ui-ux-production-standard.md`.

## Anti-Patterns

- Random colors or gradients.
- One-off card/table/form styles.
- Mixed icon systems.
- Oversized hero-like UI inside admin tools.
- Text too large for compact panels.
- Style choices that reduce Japanese readability, focus, or exam trust.

## Output Checklist

- reused components listed
- new pattern justified or avoided
- token/classes used
- visual consistency risk noted
