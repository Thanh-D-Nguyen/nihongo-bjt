# One Task One PR Rule

Each cycle must behave like one production PR:

- one task
- one owner agent
- limited files
- tests
- summary
- risks
- next task

Do not mix unrelated modules in one cycle.

## Owner rule

The owner agent is responsible for:
- scope control
- implementation or planning output
- changed-file summary
- verification commands
- handoff notes

Specialist reviewers can be added, but they do not broaden the task.

## File scope rule

Before editing, identify likely files. If implementation needs unrelated files, stop and update the cycle scope before continuing.

## PR-style closeout

End each cycle with:
- what changed
- why it changed
- how it was verified
- known risks
- recommended next task
