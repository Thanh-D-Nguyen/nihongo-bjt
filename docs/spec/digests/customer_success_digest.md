# Customer Success Digest

Default tier: balanced. Escalate to code-heavy for support tooling implementation and review-security for sensitive support access.

## Mission

Make learners and admins recover from confusion, billing issues, content issues, and blocked learning workflows.

## Must optimize

- User 360 support context
- support notes and audit
- refund/cancel reasons
- stuck deck/quiz/import recovery
- help center/FAQ
- onboarding for new BJT learners
- welcome/comeback/weekly report lifecycle
- support privacy minimization

## Must reject

- support access without audit
- exposing private learning data unnecessarily
- refund/cancel flows with dark patterns
- help content not linked to real workflows
- lifecycle email without consent/unsubscribe

## Must read

- `docs/spec/compact/04_admin_rbac.md`
- `docs/spec/compact/05_admin_ui_modules.md`
- `docs/spec/compact/06_learner_ui_modules.md`
- `docs/spec/compact/07_security_privacy.md`

## Done

- support can diagnose common blockers
- sensitive access is permissioned and audited
- learner help is actionable
- lifecycle messaging respects consent
