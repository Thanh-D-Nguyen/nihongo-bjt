# NihonGo Daily Magazine — Design Spec

## Overview

Auto-generated daily/weekly content that blends entertainment with Japanese learning. Content is generated by scheduled cron jobs using AI + real data APIs, stored in PostgreSQL, served via ISR-cached pages, and integrated with the existing learning assist layer.

## MVP Content Types (5)

| Type | widgetKind | Schedule | Data Source | Output Format |
|------|-----------|----------|-------------|---------------|
| Daily Vocab | `magazine_vocab` | 06:00 daily | Seasonal calendar + AI | 3-5 words + sentences + mini quiz |
| Weather | `magazine_weather` | 06:00 daily | JMA API (気象庁) + AI | JP weather report + vocab highlights |
| Horoscope | `magazine_horoscope` | 06:00 daily | AI generated | 12 zodiac fortunes in JP + grammar |
| Loto Prediction | `magazine_loto` | Mon+Thu 18:00 | Historical data + AI | Lucky numbers + kanji/数字 vocab |
| BJT Phrase | `magazine_bjt_phrase` | 07:00 daily | BJT corpus + AI | 1 phrase + dialogue + quiz |

## Architecture

### Builds on Existing Infrastructure

- **Schema:** Extends `daily` PostgreSQL schema (already has `DailyContentItem`, `DailyLearningExtraction`)
- **Cron:** Uses `@nestjs/schedule` `@Cron` decorators (same as existing `PushNotificationCron`)
- **Learning:** Reuses `DailyLearningExtraction` for vocab/kanji/grammar extraction
- **Reading Assist:** Existing tap-to-translate layer works on any Japanese text component

### Database Schema (new tables in `daily` schema)

```sql
-- Extends existing daily schema
-- magazine_article is more structured than DailyContentItem for long-form content

CREATE TABLE daily.magazine_article (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  slug VARCHAR(200) UNIQUE NOT NULL,
  widget_kind VARCHAR(64) NOT NULL,  -- magazine_vocab, magazine_weather, etc.
  content_date DATE NOT NULL,
  locale VARCHAR(16) NOT NULL DEFAULT 'vi',
  
  -- Display
  title_jp TEXT NOT NULL,
  title_vi TEXT NOT NULL,
  summary_jp TEXT,
  summary_vi TEXT,
  cover_image_url TEXT,
  
  -- Structured content (varies by type)
  content_json JSONB NOT NULL DEFAULT '{}',
  
  -- Learning metadata
  jlpt_level VARCHAR(8),  -- N5, N4, N3, N2, N1
  
  -- Generation metadata
  source_data_json JSONB DEFAULT '{}',
  ai_model VARCHAR(64),
  generation_cost_tokens INT,
  
  -- SEO
  seo_title TEXT,
  seo_description TEXT,
  og_image_url TEXT,
  
  -- Status
  status VARCHAR(32) NOT NULL DEFAULT 'draft',
  published_at TIMESTAMPTZ,
  expires_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  
  CONSTRAINT uq_magazine_kind_date_locale UNIQUE (widget_kind, content_date, locale)
);

CREATE TABLE daily.magazine_vocab_item (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  article_id UUID NOT NULL REFERENCES daily.magazine_article(id) ON DELETE CASCADE,
  word_jp TEXT NOT NULL,
  reading TEXT NOT NULL,
  meaning_vi TEXT NOT NULL,
  pos VARCHAR(32),  -- noun, verb, adjective, etc.
  jlpt_level VARCHAR(8),
  sentence_jp TEXT,
  sentence_vi TEXT,
  display_order INT DEFAULT 0,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE daily.magazine_quiz (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  article_id UUID NOT NULL REFERENCES daily.magazine_article(id) ON DELETE CASCADE,
  question_jp TEXT NOT NULL,
  question_vi TEXT,
  quiz_type VARCHAR(32) NOT NULL DEFAULT 'multiple_choice',  -- multiple_choice, fill_blank, true_false
  options JSONB NOT NULL DEFAULT '[]',
  correct_answer TEXT NOT NULL,
  explanation_jp TEXT,
  explanation_vi TEXT,
  display_order INT DEFAULT 0,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE daily.magazine_user_read (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL,
  article_id UUID NOT NULL REFERENCES daily.magazine_article(id) ON DELETE CASCADE,
  read_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  quiz_score INT,
  quiz_total INT,
  vocab_saved_count INT DEFAULT 0,
  time_spent_seconds INT,
  
  CONSTRAINT uq_magazine_user_article UNIQUE (user_id, article_id)
);

CREATE INDEX idx_magazine_article_public ON daily.magazine_article(status, content_date DESC, locale);
CREATE INDEX idx_magazine_article_kind ON daily.magazine_article(widget_kind, status, content_date DESC);
CREATE INDEX idx_magazine_user_read_user ON daily.magazine_user_read(user_id, read_at DESC);
```

### Content Generation Pipeline

```
Cron triggers →
  1. Fetch real data (JMA weather, loto history, seasonal calendar)
  2. Build prompt from template + real data + target JLPT level
  3. Call AI API (OpenAI/Claude) with structured output schema
  4. Validate JP content (readings check, basic grammar)
  5. Extract vocab/quiz items from generated content
  6. Save to DB (article + vocab_items + quiz)
  7. Set status = 'published'
  8. Trigger notification (optional)
```

### API Endpoints

```
GET  /api/magazine                    # List articles (paginated, filterable by type)
GET  /api/magazine/:slug              # Get article detail (public, no auth required)
GET  /api/magazine/today              # Today's articles
POST /api/magazine/:slug/read         # Mark as read + submit quiz answers
POST /api/magazine/:slug/save-vocab   # Save vocab items to user's flashcard deck

# Admin
GET    /api/admin/magazine            # List all (incl. draft)
POST   /api/admin/magazine/generate   # Trigger manual generation
PUT    /api/admin/magazine/:id        # Edit article
DELETE /api/admin/magazine/:id        # Delete
POST   /api/admin/magazine/:id/regenerate  # Regenerate with AI
```

### Frontend Routes

- `/magazine` — Archive page with filters (type, date range). ISR, revalidate 1h.
- `/magazine/[slug]` — Article detail. Public (no auth), ISR. Full SEO meta.
- Homepage "Hôm nay" widget — Shows today's articles as cards.

### Learning Integration (MVP)

1. **Tap-to-translate:** Reuse existing Reading Assist Layer. Japanese text in articles renders with `<JapaneseText>` component.
2. **Mini quiz:** 2-3 questions at end of each article. Multiple choice. Show explanation on answer.
3. **Save vocab:** Button to save highlighted words to user's flashcard deck (calls existing flashcard API).
