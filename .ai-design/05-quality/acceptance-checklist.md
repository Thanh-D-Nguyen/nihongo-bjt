# Acceptance Checklist

Before finalizing any UI task:
- [ ] Follows Refined style.
- [ ] Uses existing components/tokens.
- [ ] Handles loading state.
- [ ] Handles empty state.
- [ ] Handles error state.
- [ ] Handles disabled state.
- [ ] Mobile behavior checked.
- [ ] Keyboard/focus checked.
- [ ] i18n respected.
- [ ] No fake success or placeholder production logic.
- [ ] No unrelated refactor.
