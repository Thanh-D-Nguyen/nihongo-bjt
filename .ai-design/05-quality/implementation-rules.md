# Implementation Rules

## React/Next.js
- Prefer server/client boundary clarity.
- Do not create unnecessary client components.
- Keep components small and reusable.
- Preserve route and locale conventions.
- Avoid hardcoded strings when i18n pattern exists.

## CSS/Tailwind
- Use design tokens/classes consistently.
- Avoid one-off arbitrary values unless justified.
- Prefer semantic component variants.

## Data
- Do not fake production endpoints.
- Use loading states while awaiting real data.
